## Code of Conduct

Please be respectful and constructive in all interactions.  
We are committed to providing a welcoming and inclusive environment for everyone.

---

## How to Contribute

- **Bug Reports**: Open a GitHub Issue with a clear description and reproduction steps.
- **Feature Requests**: Open a GitHub Issue describing the motivation and proposed behaviour.
- **Code Contributions**: Fork the repository, make your changes on a feature branch, and open a Pull Request.
- **Documentation Improvements**: Corrections, clarifications, and translations are all welcome.

---

## Development Setup

1. Fork and clone the repository.
2. Install the required dependencies as described in the README.
3. Create a new branch for your change:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Make your changes and commit them with clear, descriptive messages.
5. Push your branch and open a Pull Request against `main`.

---

## Submitting a Pull Request

- Ensure your branch is up to date with `main` before opening a PR.
- Keep PRs focused — one logical change per PR is preferred.
- Provide a clear description of *what* the change does and *why* it is needed.
- Link any related issues in the PR description (e.g. `Closes #42`).
- Be responsive to review feedback; address comments promptly.
