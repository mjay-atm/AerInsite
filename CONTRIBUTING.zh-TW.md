## 行為準則

請在所有互動中保持尊重與建設性的態度。  
我們致力於為每位參與者提供友善且具包容性的環境。

## 如何貢獻

- **錯誤回報**：開啟 GitHub Issue，並提供清楚的描述與重現步驟。
- **功能請求**：開啟 GitHub Issue，說明需求動機與預期行為。
- **程式碼貢獻**：Fork 本倉庫，在功能分支上進行修改，再開啟 Pull Request。
- **文件改善**：歡迎修正、說明補充及翻譯等各類文件貢獻。

---

## 開發環境設置

1. Fork 並 Clone 本倉庫。
2. 依照 README 的說明安裝所需依賴套件。
3. 為您的修改建立新分支：
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. 進行修改，並以清楚、具描述性的訊息提交。
5. 推送分支並對 `main` 分支開啟 Pull Request。

---

## 提交 Pull Request

- 開啟 PR 前，請確認您的分支已更新至最新的 `main`。
- 請保持 PR 聚焦——每個 PR 建議只包含一個邏輯變更。
- 請清楚說明此變更的**內容**與**原因**。
- 在 PR 描述中連結相關 Issue（例如 `Closes #42`）。
- 請積極回應審查意見，並及時處理回饋。

## 貢獻者許可協議（CLA）

透過向本倉庫提交 Pull Request (PR)，即表示您同意以下條款：

1. **版權授予**  
   您授予本專案作者永久、全球性、非排他且無償的版權許可，以使用、重製及分發您的貢獻。

2. **授權靈活性**  
   您知悉並同意，專案作者保有更改專案授權條款（例如為商業合作夥伴提供雙重授權）或在未來更新開源授權版本的權利。

3. **原創性保證**  
   您保證所提交的貢獻為您的原創作品，或您已獲得合法授權並能在本條款下提交。

如您對上述條款有任何疑問，請在提交貢獻前先開啟一個 Issue 進行討論。
